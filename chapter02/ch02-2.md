# 02-2. 모듈 이해하기

- 타입스크립트에서의 ts 파일은 모듈(module)이라고 한다.
- ts 파일 하나로 구현해도 되지만, 보통 코드 관리와 유지보수를 위해 모듈마다 고유한 기능을 구현하는 방식으로 소스코드를 분할한다. = 모듈화(modulization)
- 타입스크립트는 모듈을 제공하는 쪽에서 `export`를, 모듈의 기능을 이용하는 쪽에서 `import`를 사용하는 키워드를 제공한다.

`export`는 function, interface, class, type, let, const 앞에 붙일 수 있다.
```js
export interface IPerson {
    name: string
    age: number
}
export const makePerson = (name: string, age: number) => {
    ...   
}
```

`import`의 기본적인 형태는 다음과 같다.<br>
ts 파일을 import할 때는 보통 확장자 `.ts`를 생략한다.
```js
// import { 심벌 목록 } from '파일의 상대 경로'
import { IPerson, makePerson } from './person/Person'
```

`as` 키워드를 함께 사용하여 지정한 심벌로 접근할 수 있다.
```js
// import * as 심벌 from '파일 상대 경로'
import * as U from '../utils/makeRandomNumber'

U.makeRandomNumber()
```

타입스크립트는 자바스크립트와 호환을 위해 `export default` 구문을 제공한다.<br>
- 모듈이 보내는 기능 중 오직 1개에만 붙일 수 있다.
- `import` 문으로 불러올 때 중괄호 `{}` 없이 사용할 수 있다.
```js
export default class Person implements IPerson {
    ...
}
export const makePerson = (name: string, age: number) => {
    ...
}
```
```js
import Person, { makePerson } from './person/Person'
```

외부 패키지의 기능을 사용하는 경우, `node_modules` 디렉터리에 있기 때문에 경로에서 `./` 등을 생략한 채 사용한다.
```js
import Chance from 'chance'
import * as R from 'ramda'
```
