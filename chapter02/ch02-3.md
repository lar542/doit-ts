# 02-3. tsconfig.json 파일 살펴보기

tsc 컴파일러는 컴파일 옵션과 대상 파일 목록 2가지를 입력받는다.
```bash
tsc [options] [file...]
```
생성한 tsconfig.json 파일 구성을 보면<br>
`compilerOptions` 항목은 tsc 명령 형식에서 옵션을 나타내고, `include` 항목은 컴파일 대상 파일 목록을 나타낸다.
```json
{
  "compilerOptions": {
    "ignoreDeprecations": "6.0",
    "module": "commonjs",
    "esModuleInterop": true,
    "target": "es5",
    "moduleResolution": "node",
    "rootDir": "src",
    "outDir": "dist",
    "baseUrl": ".",
    "sourceMap": true, 
    "downlevelIteration": true,
    "noImplicitAny": false,
    "paths": {"*": ["node_modules/*"]}
  },
  "include": ["src/**/*"] // 컴파일 대상 파일로 포함
}
```
👉 [최신 옵션 가이드 보기](https://aka.ms/tsc)

## module
- TS 코드가 컴파일되어 만들어진 ES5 JS 코드는 웹 브라우저와 nodejs 양쪽에서 모두 동작해야 하는데, 이 둘은 동작 방식이 다르다.
- 해당 옵션은 동작 대상 플랫폼에 따라 그에 맞는 모듈 방식으로 컴파일하려는 목적으로 설정한다.
- `commonjs`는 nodejs에서 동작함을 의미한다.

## moduleResolution
nodejs에서 동작하는 것으로 설정했으니, 해당 옵션은 `node`로 설정한다.

## target
트랜스파일할 대상 JS의 버전을 설정

## baseUrl과 outDir
- 트랜스파일된 ES5 JS 파일을 저장할 디렉터리를 설정
- tsc는 tsconfig.json 파일이 있는 디렉터리에서 실행되므로, 현재 디렉터리를 의미하는 `.`을 baseUrl로 설정하고, outDir는 baseUrl 기준으로 하위 디렉터리 이름을 뜻한다.

## paths
- `import` 문에서 `from` 부분을 해석할 때 찾아야하는 디렉터리를 설정
- 외부 패키지인 경우 `node_modules` 디렉터리에서 찾아야 하므로 `node_modules/*`도 포함했다.

## esModuleInterop
오픈소스 JS 라이브러리 중에는 웹 브라우저에서 동작한다는 가정으로 만들어진 것이 있는데, 이들은 웹 브라우저에서의 동작 방식인 `amd` 방식을 전제로 구현된 라이브러리이므로 해당 라이브러리가 동작하기 위해 true로 설정한다.

## sourceMap
- 트랜스파일 디렉터리에 `.js` 파일 외에도 `.js.map` 파일이 생성되게  true로 설정.
- 이 소스맵 파일은 변환된 JS 코드가 TS 코드의 어디에 해당하는지를 알려주는 것으로, 주로 디버깅할 때 사용된다.

## downlevelIteration
generator라는 JS 구문을 동작하기 위해 true로 설정

## noImplicitAny
- false : 타입을 명시하지 않은 경우 암시적으로 any 타입을 설정한 것으로 간주한다.
- true : 타입 미지정 시 "암시적으로 any 형식이 포함됩니다." 라는 직관적이지 않은 오류 메시지가 발생한다.
