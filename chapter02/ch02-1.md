# 02-1. 타입스크립트 프로젝트 만들기

## package.json
- 타입스크립트 개발은 nodejs 프로젝트를 만든 다음, 개발 언어를 타입스크립트로 설정하는 방식으로 진행한다.
- nodejs 프로젝트는 디렉터리를 하나 만들고 여기에 `package.json` 파일을 만드는 것으로 시작한다.
    - `package.json` : nodejs가 관리하는 패키지 관리 파일로, 프로젝트 정보와 관련 패키지를 기록한다.
    - 다음은 `package.json` 파일을 생성하는 명령어 이다.
        ```bash
        npm init --y
        ```

### 패키지 설치 명령 옵션
`package.json` 파일을 만들었으면, 프로젝트 구현에 필요한 오픈소스 패키지를 `npm install` 또는 `npm i` 명령어로 설치한다.<br>
다음은 패키지 설치 명령의 옵션이다.

| npm i 옵션 | 단축 명령 | 의미 |
|------|------|------|
| `--save` | `-S` | 프로젝트를 **실행할 때** 필요한 패키지로 설치한다. 패키지 정보가 `dependencies` 항목에 등록된다. |
| `--save-dev` | `-D` | 프로젝트를 **개발할 때만** 필요한 패키지로 설치한다. 패키지 정보가 `devDependencies` 항목에 등록된다. |

타입스크립트 프로젝트는 보통 typescript와 ts-node 패키지를 설치한다.<br>
다음은 프로젝트 내에서만 패키지를 설치하는 명령어 이다.
```bash
npm i -D typescript ts-node
```

- JS 컴파일러는 `a => a + 1`과 같은 코드를 동작시킬 수 있지만, TS 컴파일러는 `(a: number): number => a + 1`처럼 타입이 명시적으로 설정되어 있어야만 코드가 문법에 맞게 작성되었는지를 검증해 코드를 동작시킨다.
- 따라서 JS로 개발된 chace, ramda와 같은 라이브러리들은 추가로 타입 라이브러리들을 제공해야 한다.
    - `@types/`가 앞에 붙는 타입 라이브러리들은 항상 `index.d.ts`라는 파일을 가지고 있으며, TS 컴파일러는 이 파일을 바탕으로 해당 라이브러리가 제공하는 함수들을 올바르게 사용했는지 검증한다.
- 타입스크립트는 웹 브라우저나 nodejs가 기본으로 제공하는 타입들의 존재도 그냥 알지 못하기 때문에 사용하려면 타입 라이브러리를 설치해야 한다.
    - `Promise`와 같은 타입을 사용하려면 `@types/node`라는 패키지를 설치해야 한다.
        ```bash
        npm i -D @types/node
        ```

## tsconfig.json
- 타입스크립트 프로젝트는 `tsconfig.json` 파일이 있어야 한다. 이 파일은 다음 명령어로 생성할 수 있다.
    - `tsconfig.json` : 타입스크립트 컴파일러의 설정 파일
```bash
tsc --init
```

만약 타입스크립트 컴파일러(tsc)를 글로벌 설치하지 않은 상태에서 프로젝트 내에서만 tsc 명령어를 사용하려면, `package.json` 파일의 `scripts` 항목에 `"tsc": "tsc"`를 추가하고 다음 명령어로 `tsconfig.json` 파일을 생성한다.
```bash
npm run tsc -- --init
```
그리고 생성된 파일을 열어 프로젝트에 필요한 옵션만 설정하면 된다.

```json
{
  "compilerOptions": {
    "ignoreDeprecations": "6.0", // deprecated 무시
    "module": "commonjs",
    "esModuleInterop": true,
    "target": "es5",
    "moduleResolution": "node",
    "rootDir": "src",
    "outDir": "dist",
    "baseUrl": ".",
    "sourceMap": true,
    "downlevelIteration": true,
    "noImplicitAny": false,
    "paths": {"*": ["node_modules/*"]}
  },
  "include": ["src/**/*"] // 해당 디렉토리에 모든 TS 파일이 있음
}
```
참고로 시작 소스 파일명이 `index`인 경우, 파일명을 생략하고  `ts-node src` 처럼 실행할 수 있다. 이 때문에 프로젝트의 시작 함수(엔트리 함수)가 있는 소스 파일명은 보통 `index`로 짓는다.

개발할 때는 `ts-node`를 사용하지만, **개발이 완료되면** TS 소스코드를 ES5 JS 코드로 변환해 `node`로 실행해야 한다. 이를 위해 `package.json` 파일의 `scripts` 항목에 다음과 같이 `dev`와 `build` 명령을 추가한다.
```json
{
  "name": "ch02-1",
  "version": "1.0.0",
  "description": "",
  "main": "src/index.js",
  "scripts": {
    "dev": "ts-node src", // 개발 중에 src 디렉터리에 있는 index.ts 파일을 실행하는 용도
    "build": "tsc && node dist", // 개발 완료 후 배포하기 위해 dist 디렉터리에 ES5 JS 파일을 생성하는 용도
    "tsc": "tsc"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "devDependencies": {
    "@types/node": "^25.5.0",
    "ts-node": "^10.9.2",
    "typescript": "^6.0.3"
  }
}
```
