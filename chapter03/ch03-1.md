# 03-1. 타입스크립트 변수 선언문

TS는 JS와 호환성을 위해 JS 타입은 물론 그에 대응하는 TS 타입도 제공한다.<br>
다음은 JS와 TS의 기본 타입 이다.
| 유형 | 자바스크립트 타입 | 타입스크립트 타입 |
|:-----:|:-----:|:-----:|
| 수 | Number | number |
| 불리언 | Boolean | boolean |
| 문자열 | String | string |
| 객체 | Object | object |

ES5 JS에서 변수 선언 시 사용했던 `var` 키워드는 사용하지 않도록 권고하고, `let`와 `const` 키워드를 도입했다.<br>
- `let`으로 선언한 변수는 그 값이 변경될 수 있다.
- `const`로 변수 선언 시에는 반드시 초기화해야 하고, 변수 값이 절대 변하지 않는다.

TS는 다음과 같은 형태로 타입을 명시할 수 있다. = 타입 주석(type annotation)<br>
그런데 TS는 JS와 달리 `let`으로 선언한 변수 값은 타입 주석으로 명시한 타입에 해당하는 값으로만 바꿀 수 있다.
```ts
let 변수 이름: 타입 = [= 초깃값]
const 변수 이름: 타입 = 초깃값

let n: number = 1
n = 'a' // 타입 불일치 오류 발생
```

그런데 TS는 JS와의 호환성을 위해 타입 주석을 생략할 수 있는데,<br>
이 경우 컴파일러는 대입 연산자 `=` 오른쪽 값에 따라 변수 타입을 지정한다. = 타입 추론(type inference)<br>
즉, 컴파일러가 초기 값에 따라 타입 추론하기 때문에 각 변수는 초기 값에 해당하는 타입으로 지정되므로, 각 변수는 해당 타입의 값으로만 저장할 수 있다.
```ts
let n = 1       // number로 판단
let b = true    // boolean으로 판단
let s = 'hello' // string으로 판단
let o = {}      // object로 판단
```

TS는 JS와 호환을 위해 `any` 타입을 제공하는 데,<br>
이는 타입과 무관하게 어떤 종류의 값도 저장할 수 있다.
```ts
let a: any = 0
a = 'hello'
a = true
a = {}
```

JS에서 `undefined`는 값이다. 변수를 초기화하지 않으면 해당 값을 가진다.<br>
그러나 TS에서 `undefined`는 값이면서, 타입이기도 하다.<br>
다음 코드를 보면 `undefined` 타입으로 명시한 변수에 1을 저장 시 오류가 발생한다.
```ts
let u: undefined = undefined
u = 1 // 오류 발생
```
타입스크립트의 타입 계층도를 보면 `any`는 모든 타입의 최상위 타입이고, `undefined`는 모든 타입의 최하위 타입이다.<br>
이 때문에 `undefined`의 상위 타입인 number 타입을 저장하니 오류가 발생한다.

| 최상위 |  |  | 최하위 |
|-----|-----|-----|-----|
| any | number, boolean, string  |  |  |
|  | object | interface, class |  |
|  |  |  | undefined |
