# 03-2. 객체와 인터페이스

- 앞서 타입스크립트의 타입 계층도에서 `object` 타입은 `interface`와 `class`의 상위 타입 이다.
- `object` 타입으로 선언된 변수는 number, boolean, string 타입 값을 저장할 수는 없지만, 다음처럼 속성 이름이 다른 객체를 모두 자유롭게 담을 수 있다.
- `object` 타입은 마치 `any` 타입처럼 동작한다. 이를 통해 타입스크립트의 `interface`는 이렇게 동작하지 않게 하려는 목적으로 고안되었다.
```ts
let o: object = { name: 'Jack', age: 32 }
o = { first: 1, second: 2 }
```

## 인터페이스
- TS는 객체의 타입을 정의할 수 있게 하는 `interface` 키워드를 제공한다.
- 객체의 타입을 정의하는 것이 목적이므로 객체를 의미하는 중괄호 `{}`로 속성과 속성의 타입 주석을 나열하는 형태로 사용한다.
- 즉, 정의한 속성이 있는 객체만 유효하도록 객체의 타입 범위를 좁히는 데 있다.
```ts
interface IPerson {
    name: string
    age: number
}
```

## 선택 속성 (optional property)
어떤 속성은 반드시 있어야 하지만, 어떤 속성은 있어도 되고 없어도 되는 형태로 만들 수 있다. = 선택 속성(optional property)
```ts
interface IPerson2 {
    name: string    // 필수 속성
    age: number     // 필수 속성
    etc?: boolean   // 선택 속성
}
```

## 익명 인터페이스 (anonymous interface)
- TS는 `interface` 키워드를 사용하지 않고 이름 없는 인터페이스를 만들 수 있다. = 익명 인터페이스(anonymous interface)
- 주로 함수를 구현할 때 사용한다.
```ts
// 익명 인터페이스로 객체 선언
let ai: {
    name: string
    age: number
    etc?: boolean
} = { name: 'Jack', age: 32 }

// 인명 인터페이스로 함수 구현
function printMe(me: { name: string, age: number, etc?: boolean }) {
    ...
}
```
