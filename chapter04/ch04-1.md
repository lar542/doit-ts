# 04-1. 함수 선언문

- JS에서 함수는 function 함수와 [화살표 함수] 2가지가 있다.
- TS 함수 선언문은 JS 함수 선언문에서 매개변수와 함수 반환값(return type)에 타입 주석을 붙이는 형태로 구성된다.
    - parameter(매개변수) : 함수 선언문에서 함수 이름 뒤 괄호 안에 선언하는 변수
    - argument(인수 혹은 인자) : 함수 호출 시 전달하는 값
```ts
function add(a: number, b: number): number { // 매개변수
    return a + b
}

let result = add(1, 2) // 인수
```

## 매개변수와 반환값의 타입 주석 생략
함수 선언문에도 매개변수와 반환 값에 대한 타입 주석을 생략할 수 있지만, 변수 때와는 달리 함수의 구현 의도를 알기 어렵기 때문에 바람직하지 않다.

## 함수 시그니처 (function signature)
함수의 타입 = 함수 시그니처 (function signature)
```ts
(매개변수1 타입, 매개변수2 타입[, ...]) => 반환값 타입
```

다음은 printMe 함수의 시그니처 `(string, number) => void`를 나타낸다.
```ts
let printMe: (string, number) => void = function (name: string, age: number): void {}
```

## type 키워드로 타입 별칭(type alias) 만들기
타입스트립트가 제공하는 `type` 키워드는 기존에 존재하는 타입을 이름만 바꿔서 사용할 수 있게 해준다. 이런 기능을 타입 별칭(type alias)이라고 한다.
```ts
type 새로운 타입 = 기존 타입
```

- 다음은 `(string, number) => void` 함수 시그니처를 stringNumberFunc 라는 이름으로 타입 별칭을 만든 예시 이다.
- 함수의 타입, 즉 함수 시그니처를 명시하면 매개변수의 개수나 타입, 반환 타입이 다른 함수를 선언하는 잘못을 미연에 방지할 수 있다.
```ts
type stringNumberFunc = (string, number) => void

let f: stringNumberFunc = function(a: string, b: number): void {}
let g: stringNumberFunc = function(c: string, d: number): void {}
```

## 선택적 매개변수 (optional parameter)
인터페이스에 선택 속성이 있는 것처럼, 함수의 매개변수도 선택적 매개변수(optional parameter)가 있다.
```ts
function fn(arg1: string, arg?: number): void {}
```

선택적 매개변수가 있는 함수의 시그니처는 다음처럼 표현할 수 있다.
```ts
type OptionalArgFunc = (string, number?) => void
```
